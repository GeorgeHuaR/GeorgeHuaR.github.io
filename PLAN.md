

# V7.3 编辑器开发

[功能定位]
- 轻量级本地待办事项编辑器，纯前端 localStorage 缓存，无需后端服务

[MVP功能细节]
- 多清单管理：新建/重命名/删除清单
  - 删除清单时直接删除内部TODO，不迁移
- 待办 CRUD：添加/编辑/删除/勾选完成（可以参考主页“分类管理”的逻辑，直接行内编辑，不考虑移动端交互）  
  - 勾选完成样式：划掉+变灰，通过过滤按钮切换显隐
  - 添加输入框：顶部固定输入框
  - 编辑重命名：行内编辑，双击进入编辑状态
- 优先级：红黄蓝
- 排序：创建时间/优先级， 次级排序按创建时间降序（最新的在上）
- 过滤：进行中/全部
- 数据管理：独立 localStorage key editorTodoData （无需js导入导出）

[架构关键点]
- 事件模型：单次事件委托（click/dblclick/change/keydown 绑定在容器上，持久存活），规避重复绑定问题
- CSS 加载:init() 动态注入 <link>，cleanup() 移除
- 交互反馈:复用浏览器原生 confirm/prompt/alert，不依赖 CoreModules 弹窗（保持轻量，目标为可选独立页面）

- 实测刷新后筛选默认为“进行中”，建议改为“全部”
- TODO List单条样式
  - 复选框太小()
  - 删除按钮和等级按钮太近，且等级按钮太小
  - 双击打开编辑框太短，本身内容区域很长，但编辑框很短
